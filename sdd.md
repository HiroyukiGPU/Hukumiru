# 服装提案アプリ 仕様書（GitHub Pages版・高度設計版）

## 1. 概要

本システムは、ユーザーの現在地に基づく気象データを入力とし、時間帯（朝・昼・夜）ごとに最適な服装選択を提示する意思決定支援型Webアプリケーションである。

本アプリケーションの中核は、環境要因（気温・湿度・風速）を統合した指標としての体感温度を導出し、それに基づく分類アルゴリズムにより服装提案を行う点にある。体感温度の算出には、ミスナールの計算式（改良版）を採用する。

## 2. 技術構成

本システムは完全クライアントサイドで動作し、以下の構成要素から成る。

フロントエンド：HTML5 / CSS3 / JavaScript（ES6準拠）

ホスティング：GitHub Pages

外部API：Open-Meteo（REST API, APIキー不要）

永続化層：LocalStorage

位置情報取得：Geolocation API（W3C標準）

本構成により、サーバーサイドを介さない軽量かつ可搬性の高いアーキテクチャを実現する。

## 3. 機能要件

### 3.1 位置情報管理

初回アクセス時にGeolocation APIを用いて緯度・経度を取得し、LocalStorageに永続化する。以降のアクセスでは再取得を行わず、保存済みデータを再利用することでAPIコールの冗長性を排除する。

### 3.2 気象データ取得

Open-Meteo APIを利用し、1時間単位の以下の環境パラメータを取得する。

気温（temperature_2m）

相対湿度（relativehumidity_2m）

風速（wind_speed_10m）

これらはすべて体感温度算出の入力変数として扱われる。

### 3.3 時間帯モデリング

本システムでは1日を3つの離散時間帯に分割する。

#### デフォルト設定

朝：5:00〜10:00

昼：10:00〜16:00

夜：16:00〜23:00

#### カスタム設定

ユーザーはこれらの境界値を動的に変更可能であり、設定値はLocalStorageに保存される。

### 3.4 体感温度モデル

体感温度は以下の関数として定義される。

```txt
T_app = T + 0.33H - 0.7V - 4.0
```

ここで、

T：気温（℃）

H：相対湿度（%）

V：風速（m/s）

このモデルは環境要因の線形結合として実装され、知覚温度の近似値を与える。

### 3.5 時間帯別統計処理

各時間帯に属する時系列データに対し、体感温度の平均値を算出する。

```txt
T_app_avg = (Σ T_app_i) / n
```

この平均値を当該時間帯の代表値とする。

### 3.6 服装分類アルゴリズム

体感温度に基づき、以下の閾値区分により服装を決定する。

30℃以上：ノースリーブ

25〜30℃：半袖

20〜25℃：長袖

15〜20℃：カーディガン

10〜15℃：ジャケット

5〜10℃：コート

5℃未満：ダウン

これは閾値ベースの決定木として実装可能である。

### 3.7 ユーザーインターフェース

UIは情報伝達効率を最優先としたミニマル設計とする。

#### 表示要素

地域情報（簡易表記）

日付

朝・昼・夜のカード表示

#### 各カード

時間帯ラベル

平均体感温度

服装カテゴリ

視覚的アイコン

### 3.8 設定インターフェース

ユーザーは時間帯境界を調整可能であり、変更は即時反映される。設定はLocalStorageに永続化される。

### 3.9 データ更新戦略

ページロード時にAPIリクエストを実行し、最新データを取得する。キャッシュは位置情報および設定値のみに限定する。

## 4. データ構造設計

LocalStorageにおけるデータ構造は以下の通りである。

```txt
location:
{
  lat: number,
  lon: number
}

timeSettings:
{
  morning: { start: number, end: number },
  afternoon: { start: number, end: number },
  night: { start: number, end: number }
}

```

## 5. API設計

#### エンドポイント

```txt
https://api.open-meteo.com/v1/forecast
```

#### クエリパラメータ

```txt
latitude={lat}
longitude={lon}
hourly=temperature_2m,relativehumidity_2m,wind_speed_10m
timezone=Asia/Tokyo
```

## 6. 処理フロー

1. 初回判定（LocalStorage確認）
2. 位置情報取得または読込
3. APIリクエスト
4. 時系列データ取得
5. 時間帯分類
6. 体感温度算出
7. 平均値算出
8. 服装分類
9. UIレンダリング

## 7. UI/UX設計方針

認知負荷の最小化

即時意思決定支援

視覚優先設計

操作ステップの削減

## 8. 拡張可能性

本設計は以下の拡張を想定している。

降水量・天候状態の統合

機械学習によるパーソナライズ

プッシュ通知

PWA化によるオフライン対応

## 9. 制約および注意事項

完全クライアントサイド構成

外部API依存

HTTPS環境必須

## 10. 出力イメージ

朝 → 長袖昼 → 半袖夜 → カーディガン

本システムは、日常的意思決定を直感的かつ定量的に支援することを目的とする。

## 11. デザイン仕様

本節では、視認性・可読性・即時意思決定性を最大化するためのUIデザイン仕様を定義する。

### 11.1 デザイン原則

情報優先：主要情報（時間帯・体感温度・服装）を最短視線で認知可能にする

一貫性：色・余白・タイポグラフィ・アイコンの統一

階層性：重要度に応じた視覚的ヒエラルキーの明確化

最小操作：1画面完結、設定はモーダルに集約

### 11.2 レイアウト

#### グローバル構造

ヘッダー（固定）

コンテンツ（カード3枚：朝/昼/夜）

フローティング設定ボタン（右下）

#### グリッド

ベース：4pxスケール（4/8/12/16/24/32）

コンテナ最大幅：720px（中央寄せ）

カード間隔：16px

### 11.3 カラーパレット

#### 基本色

Primary：#2563EB（ブルー）

Background：#F8FAFC（ライトグレー）

Surface：#FFFFFF（カード）

Text Primary：#0F172A

Text Secondary：#475569

#### 温度レンジカラー（状態色）

=30℃：#EF4444（赤）

25–29℃：#F97316（橙）

20–24℃：#22C55E（緑）

15–19℃：#06B6D4（水色）

10–14℃：#3B82F6（青）

5–9℃：#6366F1（藍）

<5℃：#111827（濃紺）

各カードのアクセント（左ボーダー/バッジ/温度表示）に適用する。

### 11.4 タイポグラフィ

フォント：システムフォント（-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Noto Sans JP", sans-serif）

見出し（時間帯）：20–24px / 600

体感温度：28–36px / 700（最重要）

服装ラベル：16–18px / 600

補助情報：12–14px / 400

行間：1.4–1.6、字間：0–0.02em

### 11.5 カードコンポーネント仕様（WeatherCard）

#### 構造

左：状態色のアクセントバー（幅4px）

上段：時間帯ラベル + 日付（小）

中段：体感温度（大）

下段：服装ラベル + アイコン

#### スタイル

背景：#FFFFFF

角丸：16px

影：0 4px 12px rgba(0,0,0,0.06)

内側余白：16–20px

### 11.6 アイコン仕様

スタイル：フラット/ラインアイコン（統一）

サイズ：24px（テキスト横）/ 48px（強調表示）

カテゴリ対応：

ノースリーブ

半袖

長袖

カーディガン

ジャケット

コート

ダウン

### 11.7 インタラクション

カード：ホバー時に軽微な浮き上がり（transform: translateY(-2px)）

ボタン：押下時にスケール0.98

設定モーダル：フェード + スライド（200ms）

### 11.8 設定モーダル

レイアウト：中央配置のダイアログ

入力：数値入力（時間 0–23）

アクション：保存 / キャンセル

バリデーション：

0–23の範囲

重複・逆転（start < end）を禁止

### 11.9 レスポンシブ設計

モバイル（<=480px）：1カラム、カードフル幅

タブレット（481–1024px）：中央寄せ固定幅

デスクトップ（>1024px）：最大幅720px維持

### 11.10 アクセシビリティ

コントラスト比：WCAG AA以上

フォーカスリング：可視化（outline: 2px solid Primary）

代替テキスト：アイコンにaria-label付与

### 11.11 ダークモード（拡張）

Background：#020617

Surface：#0F172A

Text：#E2E8F0

状態色は彩度を維持しつつ輝度を調整

### 11.12 デザイントークン（例）

```css
--color-primary: #2563EB;
--color-bg: #F8FAFC;
--color-surface: #FFFFFF;
--radius-card: 16px;
--shadow-card: 0 4px 12px rgba(0,0,0,0.06);
--space-4: 4px;
--space-8: 8px;
--space-16: 16px;
```

### 11.13 UI状態定義

loading：スケルトン表示（カード枠 + グラデーション）

error：エラーメッセージ + 再試行ボタン

empty：データ未取得時のガイド表示

以上により、本アプリは最小限の操作で最大の意思決定支援を提供する視覚設計を実現する。
